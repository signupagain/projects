您好，我是**王宗彥**，一名前端工程師，專注於 Vue 與 Nuxt 生態系開發。雖非本科但基於個人傾向開始積極學習前端。逐步建立從 UI 實作到資料流與架構拆分的完整開發能力，把抽象架構成功落地。

## 技術選型與專案策略

初期專案，會依據設計稿的整體 UI、目標群體及產品性質，評估**開發效率**、**維護成本**與**專案生命週期**決定技術選型。比如：偏向品牌頁，我會傾向儘可能少的依賴，保持輕量和靈活；而偏向功能性較重的專案，我會選擇組件富足的庫來加速開發。讓專案複雜度依其實際需求而定，避免過度設計，確保**開發時程的精準掌控與長期維護的低成本化**。

## 狀態管理與資料流設計

狀態管理方面，根據內容建立資料流，確定組件間的依賴關係，而非盲目進行，以確保功能的正確性與可維護性。同時，會搭配**測試**，更早地發現問題，在業務需求迭代時，踏穩每一步。

另外，也**重視專案 Server 狀態**，不讓 Client 組件狀態與它長期脫勾，且依情境對應方式保持同步，避免破壞使用者體驗；也將 UI 互動與狀態邏輯分層管理，確保資料流向清晰，並且在組件內部保持單一職責，讓 UI 的變化不會影響到資料的處理邏輯，反之亦然。

## API 契約與型別安全

在 API 請求的部份，我會用 API 文件定義 Schema，將其寫成契約以凸顯 **TypeScript** 的類型優勢，確保開發的**安全性**及接口的**一致性**。這樣不僅能減少溝通成本，還能在開發過程中及早發現問題，提升整體開發效率。

## 開發理念與持續成長

總的來說，我在前端開發中會關注資料流向並分層管理，致力於打造可持續迭代的 Web 應用。且在多變的前端生態中，保持學習和適應新技術的能力，持續提升自己的專業技能。
